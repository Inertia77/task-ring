# Changelog

## 2026-07-20 — Game task links, navigation roles & README refresh

- 游戏任务编辑器的每日、指定日和本周池任务新增独立链接字段。
- 游戏执行页只为有效 HTTP/HTTPS 地址显示“打开 ↗”，链接操作与完成操作相互独立。
- 游戏配置标准化、导入导出和示例 JSON 支持任务级 `url`，并继续兼容旧版字符串任务数组。
- 本周池由多行文本框改为结构化任务行，支持新增、排序、删除和链接维护。
- 主导航将时间账本、资料库划分为冷灰描边的工具区，与前四个任务执行模块使用不同的图标、分隔和选中态。
- 今日执行、周计划池、训练饮食和游戏作战区统一使用同一套任务区域标头骨架、标题层级、高度与移动端间距。
- 重整根 README，修正目录树、模块数量和文档索引，补充编辑器、数据格式、验证与隐私说明。

## 2026-07-12 — Manual time entry

- 今日任务、周计划池、游戏作战区、时间账本与任务明细新增“补记”入口。
- 补记可填写 1–10080 分钟和完成时间，支持 15/30/60/90 分钟快速选择。
- 手动记录复用现有本机存储、Gist 同步、删除墓碑与日/周统计，并在明细中标记“手动补记”。
- 补记日期沿用 04:00 日切换和周一 04:00 周切换规则。
- 修复保存补记后页面仍被弹层滚动锁定的问题；完成日期与时刻改为响应式独立选择，手机端单列显示。

## 2026-07-11 — Command UI & completion effects

- 压缩 Header 与主导航，将今日模式、完成、剩余、关注、计时和进度合并为 Command Strip；Orbit 默认使用紧凑折叠态。
- 今日任务改为标题优先的高密度指令卡，统一逾期、遗留、高优先、计时和完成层级；周计划、游戏、时间账本与资料库同步收紧。
- 新增 `completion-effects.js` 与 `effects.css`，恢复 16 张本地角色 Cut-in，并实现子任务、普通任务、父任务/游戏大卡、每日全清四级反馈。
- 完成演出支持事件去重、三项有界队列、移动端粒子降级、图片失败文字降级、自动 DOM 清理及 `prefers-reduced-motion` 成功提示。
- 计时完成、自动完成父任务、批量遗留与游戏小项自动完成大卡均避免重复大型演出。
- 320×800 至 1920×1080 的八组视口、五个主页面再次验证无页面级横向滚动。

## 2026-07-10 — Product UI refactor

- 用 Token + 10 个功能样式层替换历史主题覆盖链。
- 新增今日分类 Accordion、持久化子任务展开和正常文档流卡片。
- 周计划改为分类筛选 + 分类 Accordion，“全部”不再铺开全部任务。
- 游戏作战区新增今日/本周指标、日期恢复、游戏选择恢复和手机单游戏详情。
- 资料库新增搜索、分组摘要、独立资料卡与上次分组恢复。
- 编辑器在手机端改为全屏，关闭按钮和底部保存操作保持可见；游戏编辑器按游戏收纳。
- 软锁 SHA-256 在 `crypto.subtle` 缺失或调用失败时自动降级。
- 任务和资料库编辑器支持无 Token 本机保存。
- 统一 Dialog、焦点恢复、焦点循环、表单标签和 Accordion `aria-expanded`。
- 删除旧主题、旧 boot/renderer 和旧版本设计文档。

历史版本的实现说明已由 `REFACTOR_REPORT.md`、`CLEANUP_REPORT.md` 与 Git 历史替代。
