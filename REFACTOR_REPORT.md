# TaskRing 重构报告

## 原项目主要问题

- 今日、周计划、游戏和资料分组默认展开过多；390px 基线中今日页高度约 5444px，默认打开 20 个 `details`。
- CSS 由多个年代主题和末尾覆盖层串联，规则重复、`!important` 密集，状态与响应式来源不清晰。
- 周计划 renderer 被多个脚本依次覆盖，并由 boot 脚本重复 `renderAll()`。
- 手机布局主要依赖桌面规则缩放，编辑器操作区和复杂字段缺少明确分区。
- 游戏日期/游戏选择、资料分组、周分类和多类 Accordion 的恢复策略不统一。
- `crypto.subtle` 存在但不可用时，软锁哈希会直接抛错，缺少异常降级。
- 任务/资料编辑器在无 Gist Token 时拒绝本机保存。
- Dialog 角色、焦点返回、焦点循环和动态表单标签不完整。

## 修改的文件

- `index.html`：加载链、Dialog 语义、表单关联、导航状态和缓存版本。
- `assets/js/app.js`：软锁降级、本机保存、启动顺序、状态恢复和重复监听清理。
- `assets/js/views/product-ui.js`：正式产品 UI renderer、Accordion/筛选持久化、Dialog 与表单可访问性。
- `assets/js/views/editor-ux.js`、`time-ledger-view.js`：保留各自单一职责。
- `assets/css/*.css`：新 Token、布局、组件、五个业务页面、编辑器与响应式体系。
- `README.md`、`docs/*`：启动、结构、资源和变更说明。

## 新页面结构

- Header：品牌、日期/周期、同步、总控和锁定。
- 主导航：桌面粘性导航；700px 以下为安全区底部导航。
- 今日：进度/紧急/计时状态 → 概览 → 分类 Accordion → 任务卡 → 正常流子任务。
- 周计划：总指标 → 分类横向筛选 → 类别摘要 → 类别 Accordion → 任务卡。
- 游戏：标题/计时/今日按钮 → 今日/本周切换 → 指标 → 日期 → 游戏选择 → 当前详情。
- 时间：今日/本周指标 → 活动计时器 → 总览/任务账/明细。
- 资料：搜索/管理 → 分组 Accordion → 独立资料卡。

## 状态管理

新增 `taskring_ui_disclosure_v1` 保存业务 Accordion。周分类、游戏日期、游戏选择、游戏模式、资料搜索和最后分组使用独立稳定 key。`renderAll()` 继续在渲染前收集滚动状态、渲染后双 RAF 恢复，不主动居中第一个 Tab。

## 响应式策略

- 1024px：双列卡片收敛为单列，Header 工具区重排。
- 700px：底部主导航、全屏编辑器、固定保存栏、纵向任务动作区、安全区和正文底部预留。
- 359px：工具栏、指标和动作区进一步单列；游戏卡隐藏次要图标但保留标题和数字。
- 所有 Grid/Flex 关键子项使用 `min-width: 0`，页面根节点使用 `overflow-x: clip` 作为最后防线。

## 锁屏修复

`sha256Hex()` 先尝试 Web Crypto；当 `crypto.subtle` 缺失或 `digest()` 拒绝时，自动使用项目内 SHA-256。`taskring2026` 的两条路径都生成同一哈希。错误密码保持锁定；正确密码、记住浏览器和手动重新上锁均已验证。

## 收纳逻辑

- 今日：含计时、逾期或高优先任务的分类优先展开；全部完成分类默认收起；用户选择覆盖默认值。
- 子任务：按任务+周期+日期保存；部分完成时首次默认展开。
- 周计划：正在计时 → 接近周目标 → 首个有任务分类；“全部”页面仍按分类 Accordion 收纳。
- 游戏：冷启动今日；日期、当前游戏和本周筛选保存；手机只呈现当前游戏详情。
- 资料：首次只展开第一组，之后恢复最后查看分组；搜索时只临时展开匹配分组。

## 无障碍改进

- 主导航使用 `aria-current`，模式 Tab 使用 `aria-selected`。
- Accordion 动态维护 `aria-expanded`。
- Dialog 使用 `role=dialog`、`aria-modal`、标题关联、焦点循环和关闭后焦点返回。
- 动态编辑器字段自动补齐可访问名称。
- 图标/状态不只依赖颜色；高优先、逾期、运行和暂停均有文字。
- `:focus-visible` 外环、44px 点击区和 `prefers-reduced-motion` 已统一。

## 实测结果摘要

- 390×844、360×800、320×800、768×1024、1024×768、1280×720、1440×900、1920×1080：五个主页面 `scrollWidth === clientWidth`。
- 1/5/12 条子任务：分别渲染 1/5/12 行；容器 `max-height: none`、`overflow: visible`，最后一行在卡片边界内。
- 游戏选择“异环”刷新后保持；指定日期和返回今日正常。
- 计时开始、暂停、继续、完成和今日完成数联动正常。
- 任务、资料、游戏编辑器在 390px 下全屏可用，关闭与保存操作可见。
- 浏览器控制策略阻止直接导航 `file://`；无 Web Crypto 路径改用从项目源代码提取函数的 Node 隔离测试验证。

## 第二轮交互与密度修正（2026-07-10）

- 总控菜单不再依赖页面绝对坐标：桌面端锚定触发按钮并自动避让视口边缘，手机端固定为底部双列菜单，始终位于底部导航上方。
- 「显示全周」恢复为任务 × 星期矩阵。任务固定纵向排列，七天横向排列，完成、遗留、逾期与锁定都有独立文字/符号；窄屏仅矩阵组件内部横向滚动。
- 资料库桌面端由两列大卡改为自适应三列紧凑卡，手机卡片操作区与正文并排；实测桌面与 390px 手机常规卡高均为 72px。
- 资料库编辑器调整为约 1:5 的分组设置 / 条目编辑比例，条目标题与链接调整为约 0.56:1.44，避免标题被左侧分组字段挤压。
- 锁屏移除默认密码明文提示；错误密码继续保持锁定，正确密码 `taskring2026` 仍可正常解锁。
- 计时悬浮条和时间账本中的暂停/继续、完成、放弃按钮改为黄底黑字、白底黑字、深底白字三种单色状态，不再出现白底白字。
- 手机端头部压为两行工具栏，今日状态改为三列紧凑指标，任务环详情横向收纳，游戏指标在 360px 以上四列、320px 两列，重复说明块在手机端隐藏。
- 320×800、360×800、390×844、768×1024 与 1280×720、1440×900、1920×1080 再次验证五个主页面；所有视口均满足 `scrollWidth === clientWidth`。
